# LLM 调用日志独立记录

- **状态**: ✅ 已完成
- **完成时间**: 2026-06-23 12:06
- **方案**: 方案1 拦截器模式（IoLlmHttpClient 层拦截）

## 上下文

用户需求：把 LLM 请求和返回结果保存到独立日志中，方便查看最近 AI 调用。

### 用户决策
- 覆盖范围：仅 Flutter 前端
- 存储方式：本地文件 JSONL
- 查看入口：新增 "AI 调用日志" 页面
- 脱敏：不脱敏，完全记录

### 关键技术发现
前端 LLM 调用入口收口在 `novel_app/lib/services/dsl_engine/llm_provider.dart` 的 `IoLlmHttpClient`，所有 AI 调用（Dify 路径 + Hermes Agent 路径）都经过 `postJson()` / `postJsonStream()`。

## 计划步骤

1. ✅ 创建数据模型 `LlmCallRecord`（`llm_call_record.dart`）
2. ✅ 实现存储服务 `LlmLogger`（`llm_logger.dart`，JSONL + 内存缓存 + 7天清理）
3. ✅ 在 IoLlmHttpClient 的 `postJson` / `postJsonStream` 加拦截（含流式响应缓冲收集）
4. ✅ 实现列表页 `LlmLogListScreen`（按状态过滤、刷新、清空、复制摘要）
5. ✅ 实现详情页 `LlmLogDetailScreen`（JSON 美化、可折叠、复制）
6. ✅ 集成入口：settings_screen.dart 添加 "AI 调用日志" ListTile
7. ✅ 初始化：main.dart 中 `LlmLogger.instance.initialize()`
8. ✅ 单元测试：9 个用例全部通过
