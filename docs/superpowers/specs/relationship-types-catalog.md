# 人物关系类型词表(Novel Builder 人物关系图)

> 本文件是人物关系图 `RelationType` 枚举的权威词表。v1 一次性铺满,加值需发版。

## 设计要点

- 关系类型为 **Dart 封闭枚举 `RelationType`**,加值需发版,故 v1 穷尽铺满(**17 大类、109 个**)。
- 每个 enum 值带字段:`forward`(正向词 / source 视角)、`reverse`(反向词 / target 视角)、`color`、`symmetric`。
- **方向语义**:关系 `(A→B, type)` 表示"A 是 B 的 [forward]";A 的出度显示 `forward`,B 的入度显示 `reverse`。
- **对称类型** `forward == reverse`(如朋友);**不对称类型** 正反词不同(如师傅↔徒弟)。
- 颜色按大类分色系,便于可视化连线着色与分组。
- 第 17 大类(成人向)**无独立开关**,统一暗色系;iOS 审核注意不要在商店截图/预览暴露。
- 冷僻关系用 `description` 自由文本兜底,不污染主类型集。

## 边界讨论:不收录为关系类型

关系枚举**只收录"持续的、相对稳定的、有明确双向语义"的关系**。以下用其他机制承载:

- **瞬时事件**(救命/背叛/赐婚/收徒/挡刀):事件本身不是关系,完成后转化为对应关系(`savior_saved`/`betrayer_betrayed`/`spouse`/`master_disciple`)。
- **阶段/状态**(初恋/现任前任/暧昧/恋人未满/已退婚):不稳定,用关系类型 + 角色备注;`ex_lover` 已覆盖前任。
- **角色定位**(主角/女主/反派BOSS/天命之子/穿越者/炮灰/工具人):角色属性字段,非关系。
- **程度修饰**(挚友vs泛交/死党闺蜜/深仇小怨):用 `strength` 强度字段表达,不拆类型,否则枚举爆炸。
- **抽象概念**(羁绊/缘分/渊源/关系复杂):太泛,必须落到具体类型。
- **势力级关系**(同盟国/敌对国/联姻家族/世仇家族):势力图层,不是人物关系;具体到人用 `faction_ally`/`blood_feud_enemy` + 备注。

## 与已有类型的边界(第 17 大类)

- vs `master_slave`(奴隶制):后者是无自由的劳动奴隶(西幻/异界设定);`sex_slave_master` 以性服务为核心。
- vs `concubine`(妾室):妾室是古代婚姻制度内的正式身份(有名分);第 17 类是制度外的占有。
- vs `mistress`(外室):外室是长期婚外情(偏经济包养);第 17 类更强调占有/奴役。
- vs `soul_contract_master`(魂契):魂契是灵魂层面主仆契约(玄幻硬设定),不一定含性。
