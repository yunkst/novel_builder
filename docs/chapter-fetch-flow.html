Chapter Content Fetch Flow

WebView-first fallback chain for uncached chapters · Reader + Preload paths

当前章节加载 (同步)

  • • UI 触发 loadChapter → 查缓存
  • • 未命中 → 走 WebView 优先 降级链
  • • 验证 content 长度 ≥ 50，写入 SQLite
  • • 渲染段落，更新阅读进度，标记已读
  • • 修改后：未验证脚本也能走 WebView

预加载任务队列 (异步)

  • • 当前章节渲染后才触发 _startPreloadingChapters
  • • filterUncachedChapters 过滤已缓存
  • • 后续章节优先入队 → 前序章节次之
  • • _processingCompleter 互斥锁防并发
  • • 30s/任务 速率限制，缓存命中 reset

关键不变量 / 注意点

  • • _isFetching 单例锁：并发 WebView 调用 → return null → 回退 API
  • • verified=0 仅作历史标记，不再禁用
  • • 任务队列是内存队列，App 关闭即清空
  • • 后端 502 时 → 走 webview 失败 → 整个章节读取失败
  • • chapter_content_provider.dart 仍走 API，是另一条独立路径
