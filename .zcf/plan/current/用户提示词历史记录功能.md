# 用户提示词历史记录功能

## 任务概述
在章节创建全屏页面（InsertChapterScreen）中添加提示词历史记录功能。
用户输入的提示词自动保存，下次创建章节时可调出历史记录复用。
重复提示词自动提升到最新，列表不会出现重复项。

## 技术方案
- 存储：SQLite 新表 `prompt_history`，数据库升级到 v22
- 架构：遵循现有 Repository + Riverpod Provider 模式
- UI：输入框 suffixIcon 历史按钮 → BottomSheet 列表
- 范围：全局共享（所有小说共用一个历史池）

## 执行步骤

### Step 1: 数据库迁移（v22）
**文件**: `lib/core/database/database_migrations.dart`
- `currentVersion` 改为 22
- `createV1Tables` 中添加 `prompt_history` 表创建
- `upgrade` 方法添加 case 22

**表结构**:
```sql
CREATE TABLE prompt_history (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  prompt_text TEXT NOT NULL UNIQUE,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
)
```
```sql
CREATE INDEX idx_prompt_history_updated_at ON prompt_history(updated_at)
```

### Step 2: 数据模型
**新文件**: `lib/models/prompt_history.dart`
- PromptHistory 类：id, promptText, createdAt, updatedAt
- fromMap / toMap / copyWith

### Step 3: Repository
**新文件**: `lib/repositories/prompt_history_repository.dart`
- 继承 BaseRepository
- 方法：
  - `addOrUpdate(String promptText)` — 插入或更新 updated_at（去重+提升）
  - `getAll({int? limit})` — 按 updated_at DESC 返回列表
  - `search(String keyword)` — 按关键词模糊搜索
  - `delete(int id)` — 删除单条
  - `deleteAll()` — 清空全部

**新文件**: `lib/core/interfaces/repositories/i_prompt_history_repository.dart`
- 抽象接口定义

### Step 4: Riverpod Provider
**文件**: `lib/core/providers/database_providers.dart`
- 添加 `promptHistoryRepository` Provider（@riverpod）

**文件**: `lib/core/providers/service_providers.dart`
- 无需修改（此功能不需要独立 Service 层，Repository 直接供 UI 使用）

### Step 5: UI — 历史提示词 BottomSheet
**新文件**: `lib/widgets/prompt_history_bottom_sheet.dart`
- 搜索框 + 可滚动列表
- 每条显示截断文本 + 相对时间（"3分钟前"/"昨天"）
- 点击选中 → 填入输入框 → 关闭 BottomSheet
- 长按或滑动删除

### Step 6: 集成到 InsertChapterScreen
**文件**: `lib/screens/insert_chapter_screen.dart`
- 手动模式 `_userInputController` 输入框（~第655行）：suffixIcon 添加历史按钮
- 大纲模式 `_userInputController` 输入框（~第770行）：suffixIcon 添加历史按钮
- 点击按钮 → 调出 PromptHistoryBottomSheet
- `_handleConfirm()` 确认时（~第209行）：保存/更新提示词到历史

### Step 7: 运行 build_runner 生成 Provider 代码
```bash
cd novel_app && dart run build_runner build --delete-conflicting-outputs
```
