# 标签（Prompt Tags）功能

## 任务概述
新增"标签"功能：用户可在创建章节或修改段落时从预置标签库中选取标签，
提交时把选中标签对应的提示词拼接到用户输入前。

标签库需要分类管理（分类可能很多），提供独立管理页面供 CRUD。
标签选择 UI 用 BottomSheet 弹出，支持分类切换和搜索。

## 涉及入口
- **创建章节**：`insert_chapter_screen.dart` 的 `_userInputController`（手动模式 + 大纲模式）
- **段落改写**：`paragraph_rewrite_dialog.dart` 的 `userInputController`
- **管理页面入口**：`settings_screen.dart` 添加菜单项
- **拼接逻辑**：在两个入口提交生成前，调用同一个 Service 方法

## 技术方案
- 存储：SQLite 新增 `prompt_tag_categories` + `prompt_tags` 两张表，数据库 v23
- 架构：Repository + Riverpod 模式
- UI：BottomSheet 多选（Chip 形式），支持分类 Tab 切换 + 搜索
- 拼接：选中标签的 `prompt_text` 用换行拼接，置于 `userInput` 之前

**表结构**：
```sql
-- 标签分类
CREATE TABLE prompt_tag_categories (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL UNIQUE,
  sort_order INTEGER NOT NULL DEFAULT 0,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
)
```
```sql
-- 标签
CREATE TABLE prompt_tags (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  category_id INTEGER NOT NULL,
  name TEXT NOT NULL,
  prompt_text TEXT NOT NULL,
  sort_order INTEGER NOT NULL DEFAULT 0,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL,
  FOREIGN KEY (category_id) REFERENCES prompt_tag_categories(id) ON DELETE CASCADE,
  UNIQUE(category_id, name)
)
```
```sql
CREATE INDEX idx_prompt_tags_category_id ON prompt_tags(category_id)
```

## 关键设计决策
1. **不绑定小说**：标签是全局资源（与提示词历史一致）
2. **拼接格式**：`<tag1 prompt>\n<tag2 prompt>\n\n<user input>`（标签间单换行，标签和用户输入间双换行）
3. **分类可能很多**：用横向 Scrollable Tab 而非普通 Tab，支持左右滑动
4. **性能**：SQLite 标签量级 100-1000 都没问题，全量加载 + 客户端过滤
